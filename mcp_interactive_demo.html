📚 Core MCP Terminology

🖥️
MCP Host
The AI application that connects to and uses MCP servers. Acts as the client in the MCP relationship.
Examples: Claude Desktop, VS Code with MCP extension, custom AI applications
📱
MCP Client
The component within the host that handles MCP protocol communication and manages connections to servers.
Role: Protocol handling, connection management, request/response processing
🔌
MCP Server
Programs that expose tools and resources through the MCP protocol. Can be standalone processes or integrated services.
Examples: File system server, database connector, API wrapper, custom business logic
🛠️
Tools
Functions that the AI can call to perform actions. Tools have schemas defining their inputs and outputs.
Examples: search_files(), send_email(), query_database(), create_calendar_event()
📄
Resources
Data sources that the AI can read from. Resources provide context and information to enhance AI responses.
Examples: file contents, database records, API responses, documentation
🔗
Protocol
The standardized communication format that enables secure, typed interactions between hosts and servers.
Features: JSON-RPC based, type safety, capability discovery, bidirectional communication

💡 Interactive Examples

📁 File System MCP Server

Scenario: AI needs to read, write, and search files on the local system

🛠️ Tools Provided:

  • read_file(path)
  • write_file(path, content)
  • list_directory(path)
  • search_files(query, path)

📄 Resources Provided:

  • File contents
  • Directory listings
  • File metadata
  • Search results
{ "host": "Claude Desktop", "server": "filesystem-mcp-server", "connection": "stdio", "capabilities": { "tools": ["read_file", "write_file", "list_directory", "search_files"], "resources": ["file:///*"] } }

🗃️ Database MCP Server

Scenario: AI needs to query and analyze data from a PostgreSQL database

🛠️ Tools Provided:

  • execute_query(sql)
  • get_schema(table)
  • list_tables()
  • explain_query(sql)

📄 Resources Provided:

  • Query results
  • Table schemas
  • Database metadata
  • Query execution plans
{ "host": "VS Code", "server": "postgres-mcp-server", "connection": "tcp://localhost:5432", "capabilities": { "tools": ["execute_query", "get_schema", "list_tables"], "resources": ["postgres://db/*"] } }

🌐 REST API MCP Server

Scenario: AI needs to interact with a third-party REST API (e.g., weather service)

🛠️ Tools Provided:

  • get_weather(location)
  • get_forecast(location, days)
  • get_alerts(location)
  • search_locations(query)

📄 Resources Provided:

  • Current weather data
  • Forecast information
  • Weather alerts
  • Location data
{ "host": "Custom AI App", "server": "weather-api-mcp-server", "connection": "https://api.weather.com", "capabilities": { "tools": ["get_weather", "get_forecast", "get_alerts"], "resources": ["weather://api/*"] } }

🔄 Multi-Server Setup

Scenario: AI agent needs access to files, database, and external APIs simultaneously

🎯 Architecture Benefits:

  • Modular: Each server handles one concern
  • Scalable: Add/remove servers as needed
  • Secure: Each server has specific permissions
  • Maintainable: Independent server lifecycles
{ "mcpServers": { "filesystem": { "command": "mcp-server-filesystem", "args": ["--path", "/workspace"] }, "database": { "command": "mcp-server-postgres", "args": ["--connection", "postgresql://localhost/mydb"] }, "weather": { "command": "mcp-server-weather", "args": ["--api-key", "${WEATHER_API_KEY}"] }, "calendar": { "command": "mcp-server-google-calendar", "args": ["--credentials", "calendar-creds.json"] } } }

🖥️ Setting Up Claude Desktop with MCP

📋 Prerequisites

  • Claude Desktop application installed
  • Python 3.8+ or Node.js 18+ (depending on your MCP server)
  • Basic command line familiarity
1

Install Claude Desktop

Download and install Claude Desktop from the official website.

# Download from:
https://claude.ai/download
2

Locate Configuration File

Find your Claude Desktop configuration file location.

# macOS:
~/Library/Application Support/Claude/claude_desktop_config.json

# Windows:
%APPDATA%\Claude\claude_desktop_config.json
3

Install MCP Server

Install an MCP server (e.g., filesystem server).

# Install filesystem MCP server:
npm install -g @modelcontextprotocol/server-filesystem

# Or using uvx:
uvx mcp-server-filesystem
4

Configure MCP Server

Add MCP server configuration to Claude Desktop.

{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["@modelcontextprotocol/server-filesystem", "/path/to/allowed/dir"]
    }
  }
}
5

Restart Claude Desktop

Close and restart Claude Desktop to load the MCP server.

✅ MCP server will now be available in Claude Desktop
6

Test the Integration

Try using MCP tools in Claude Desktop.

"Can you list the files in my project directory?"

"Please read the contents of README.md"

"Search for files containing 'API'"

🔧 Troubleshooting Tips

  • MCP server not appearing: Check the configuration file syntax and restart Claude Desktop
  • Permission errors: Ensure the specified directory paths exist and are accessible
  • Connection failures: Verify the MCP server is properly installed and the command path is correct
  • Tools not working: Check Claude Desktop logs for error messages about the MCP server
