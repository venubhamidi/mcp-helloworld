MCP Standardized Tool Definition

Each pizza place now exposes the same interface:

// Each pizza provider implements the same MCP tool
{
  "name": "order_pizza",
  "description": "Order a pizza from this restaurant",
  "inputSchema": {
    "type": "object",
    "properties": {
      "customer": { "type": "string" },
      "pizza_type": { "type": "string" },
      "size": { "type": "string", "enum": ["small", "medium", "large"] }
    },
    "required": ["customer", "pizza_type", "size"]
  }
}

Now the AI Only Needs ONE Integration!

// Universal MCP client - works with ALL pizza providers!
class MCPClient {
  async callTool(serverUrl, toolName, params) {
    const request = {
      jsonrpc: "2.0",
      method: "tools/call",
      params: {
        name: toolName,
        arguments: params
      },
      id: 1
    };
    
    const response = await fetch(serverUrl, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(request)
    });
    
    return await response.json();
  }
}

// Use it with any pizza provider!
const client = new MCPClient();

// Order from Pizza Palace
await client.callTool(
  'mcp://pizzapalace.com',
  'order_pizza',
  { customer: 'John', pizza_type: 'pepperoni', size: 'large' }
);

// Same code works for Domino Clone!
await client.callTool(
  'mcp://dominoclone.com',
  'order_pizza',
  { customer: 'John', pizza_type: 'pepperoni', size: 'large' }
);

üîÑ JSON-RPC: The Communication Language

JSON-RPC is the underlying protocol MCP uses. It's a simple, standardized way to call functions remotely.

What is JSON-RPC?

JSON-RPC = JSON (data format) + RPC (Remote Procedure Call)

It's a protocol where you send a JSON request to call a function on a remote server, and get a JSON response back.

JSON-RPC Request Structure

{ "jsonrpc": "2.0", // Protocol version "method": "tools/call", // What function to call "params": { // Function parameters "name": "order_pizza", "arguments": { "customer": "John", "pizza_type": "pepperoni", "size": "large" } }, "id": 1 // Request ID for matching response }

JSON-RPC Response Structure

{ "jsonrpc": "2.0", "result": { // Successful result "content": [ { "type": "text", "text": "Order placed! Your large pepperoni pizza will arrive in 30 minutes. Order ID: #12345" } ] }, "id": 1 // Matches request ID }

JSON-RPC Error Response

{ "jsonrpc": "2.0", "error": { "code": -32602, "message": "Invalid params: size must be small, medium, or large" }, "id": 1 }

How JSON-RPC Solves the Problem

‚ùå Without JSON-RPC

  • REST with different endpoints
  • GraphQL with schema complexity
  • SOAP with XML overhead
  • Custom protocols everywhere
  • Parsing nightmare

‚úÖ With JSON-RPC

  • Single endpoint per server
  • Simple request/response pattern
  • Lightweight JSON format
  • Built-in error handling
  • Easy to parse and validate

Common MCP Methods Using JSON-RPC

// List available tools
{ "method": "tools/list", "params": {} }

// Call a specific tool
{ "method": "tools/call", "params": { "name": "order_pizza", "arguments": {...} } }

// List available prompts
{ "method": "prompts/list", "params": {} }

// Get a prompt
{ "method": "prompts/get", "params": { "name": "order_template" } }

üìö Breaking Down "Model Context Protocol"

ü§ñ MODEL

What it means: The AI language model (like Claude, GPT, etc.)

Pizza analogy: The customer who wants to order pizza

Example: Claude is a model that wants to order pizza for a user. Claude needs a way to communicate with different pizza restaurants.

Model: "I need to order a large pepperoni pizza for John"

üîó CONTEXT

What it means: The information and tools the model has access to

Pizza analogy: All the available pizza restaurants and their menus

Example: The model's context includes:

  • Available pizza providers (tools)
  • What each provider offers (tool schemas)
  • How to communicate with them (MCP protocol)
  • Current conversation and data (resources)
Context: "I can order from Pizza Palace, Domino Clone, or Pizza Hut Clone"

üì° PROTOCOL

What it means: The standardized rules for communication

Pizza analogy: The universal ordering system all restaurants use

Example: The protocol defines:

  • How to format requests (JSON-RPC)
  • What methods are available (tools/call, tools/list, etc.)
  • How errors are handled
  • How responses are structured
Protocol: "All requests must be JSON-RPC 2.0 format with method and params"

Putting It All Together

Model Context Protocol = A standardized way for AI models to access and interact with external tools and data

Real Example:

When you ask Claude to "order me a large pepperoni pizza," the Model (Claude) uses its Context (knowledge of available pizza ordering tools) and communicates using the Protocol (MCP/JSON-RPC) to execute the order through a standardized interface.

üí° Key Takeaways

  • Standardization: MCP provides a uniform way to connect AI models with external services
  • JSON-RPC: Simple, reliable protocol for remote function calls
  • Scalability: Add new tools without changing the core integration
  • Discoverability: Models can query what tools are available
  • Flexibility: Works across different AI models and service providers

üöÄ Why MCP Matters

Just like how USB-C standardized device charging, MCP standardizes how AI models interact with tools and services. Instead of building custom integrations for every tool, developers can build once and work with any MCP-compatible model or server. This creates an ecosystem where AI can seamlessly access thousands of tools through a single, standardized protocol.
